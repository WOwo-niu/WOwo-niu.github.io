PKI

公钥基础设施（Public Key Infrastructure）：是基于非对称密码体制（公钥密码体制）的，用来生成、管理、存储、分发和撤销数字证书的一套硬件、软件、人员、策略和过程。

PKI体系

  • 认证中心CA（Certificate Authority）：证书的签发机构
  • 注册机构RA（Registration Authority）：CA的延展，承担会多从CA处继承的管理功能
  • 证书库（Certification Library）

数字信封

信息发送端使用接收端的公钥，将一个通信密钥（对称密钥）进行加密，生成一个数字信封。

信息接收端使用自己的私钥打开数字信封，获取通信密钥，用它来解密被加密的信息。

VPN

VPN (Virtual Private Network)（虚拟专用网络）：是使用隧道技术，在公共网络创建出虚拟的、点到点安全连接的专用网络的技术。

IPSec

IPSec（IP Security）是Internet的网络层安全协议，用于在IP网络上提供数据的加密、认证和完整性保护。

IPSec中的安全组合（SA）的基本概念。

SA（Security Association）（安全组合或安全关联）：为使通信双方的认证/加密算法及其参数、密钥的一致，相互间建立的联系。

数据传输是双向的，不同方向可能存在不同的策略，因此SA是单向的，在双向通信时要建立两个SA。

三元组（源IP地址、目标IP地址、安全参数索引（SPI：用于区分不同SA））唯一地标识一个SA

安全参数索引（SPI)

认证头标AH(Authentication Head)的原理稍微了解下

  1. AH协议提供无连接的完整性、数据源认证和抗重放保护服务,不提供保密性服务
  2. AH使用消息认证码（MAC）对IP进行认证
  • 安全参数索引（Security Parameters Index）：此32比特和目的IP地址、IPSec协议（AH或ESP）组合即可==确定SA==，以确定采用的IPSec协议、操作模式、密码算法、密钥等。当建立新SA时，SPI的产生由目的系统选择。
  • 序列号（Sequence Number Field）：单调增加的32位无符号整数，利用该域==抵抗重发攻击（Replay Attack）==。每个IPSec分组依次将序列号加1.
  • 认证数据（Authentication Data Variable）：是一个长度可变的域，长度为32比特的整数倍。具体格式因认证算法而异 。该认证数据也被称为==数据报的完整性校验值==（Integrity Check Value,== ICV==）

封装安全载荷头标ESP(Encapsulating Security Payload) 的原理稍微了解下

  1. ESP提供数据保密、无连接完整性、抗重播服务
  2. ESP大都采用对称密码体制加密数据
  3. ESP使用消息认证码（MAC）提供认证服务

填充的目的：

  1. 加密算法要求明文为某个数目字节的整数倍；
  2. 32位对齐；
  3. 隐藏实际载荷长度，提供流量保密性

IPSec的传输模式与隧道模式。

  • 传输模式：在传输模式中，AH和ESP头标被插在IP头标及其他选项（或扩展头标）之后，但在传输层协议之前。它保护净荷的完整性和机密性。
  • 在隧道模式下，AH或ESP头标插在IP头标之前，另外生成一个新的IP头放在前面，隧道的起点和终点的网关地址就是新IP头的源/目的IP地址。保护整个IP分组

IKE

IKE（Internet Key Exchange）：因特网密钥交换协议，是在IPSec（Internet Protocol Security）中用于建立安全通信的协议。它是一种自动化的密钥管理协议，用于协商和交换用于加密和认证的密钥。

ISAKMP

ISAKMP（Internet Security Association and Key Management Protocol）：因特网安全组合和密钥管理协议，是用于建立和管理安全关联（Security Association，SA）和密钥的协议。它是在IPSec（Internet Protocol Security）中使用的主要协议之一，用于进行密钥交换和安全参数协商。

ISAKMP通常与IKE（Internet Key Exchange）一起使用，共同实现IPSec中的安全通信。IKE是基于ISAKMP的一种实现，它在ISAKMP的基础上进一步扩展和定义了建立和管理安全关联的过程。

iTunes备份解析

在iTunes中，所有的文件都存储在00，0a，0b···，fe，ff等目录中，目录中的文件都是一串40位的哈希值。

文件 描述
Info.plist 存储iOS设备的信息，例如内部版本号、设备名称、GUID、IMEI、安装的程序、上次备份日期、产品类型、产品版本、序列号、UDID、以及iTunes的版本等信息
Manifest.plist 存储备份文件整体的信息。
在Root键中存储备份的时间等信息
在Lockdown键中存储设备的UDID、版本号、内部代号等信息
在Applications键中存储应用程序的名称以及路径等信息
Status.plist 存储备份文件的状态，例如备份文件是否为全备份，本分的版本，UUID，日期以及快照状态等信息
Manifest.db 是一个SQLite数据库文件，Files表存储着fileID、domain、relativePath、flags等字段，其中，fileID存储的是0a，0b···fe，ff等目录中的文件名称；domain存储的是文件所属的域；relativePath存储的是文件的相对路径

Android分区结构

system分区

  • /system/app/：存放系统应用程序APK文件
  • /system/lib/：存放应用程序用到的库文件
  • /system/bin/和/system/xbin/：存储Linux命令程序
  • /system/framework/：存储Android系统运行时所需的一些框架
  • /system/etc/：存储一些配置文件
  • /system/media/：存储的是一些媒体文件，如手机铃声、壁纸等。

userdata分区

  1. /userdata/data/

存放应用程序数据，以QQ为例，在/userdata/data/com.tencent.mobileqq/目录下存储着QQ相关的数据，如聊天记录的数据库文件

  1. /userdata/media/0/

    • /Android：主要存储的是一些普通文件

    • /DICM：存储的是手机拍摄的照片、手机截图等文件

    • /Download：存储的是手机下载的文件

    • /Pictures：存储除了/DICM以外的图片文件

    • /Tencent：存储QQ、微信等腾讯应用数据，例如微信中的一些图片文件。

  2. /userdata/system/

    • /preinstall.list：该文件记录系统安装的应用程序
    • /packages.list：该文件记录应用程序的安装位置
    • /locksettings.db，gatekeeper.pattern.key，gatekeeper.pattern.key，device_policies.xml：这些文件是与锁屏密码相关的文件

  3. /userdata/misc/

    该目录中存储的是与VPN、WiFi、KeyStore等相关的配置文件。

    • wpa_supplicant.conf：文件记录了设备曾经连接的WiFi的SSID以及密码等信息
    • p2p_supplicant.conf：文件记录了设备的WiFi热点名称及其他信息

数据库操作

  • 查询所有数据库

    SHOW DATABASES;

  • 查询当前数据库

    SELECT DATABASE();

  • 创建数据库

    CREATE DATABASE [IF NOT EXISTS] 数据库名 [DEFAUL CHARSET 字符集 ] [COLLATE 排序规则];

  • 删除

    DROP DATABASE [IF EXISTS] 数据库名;

  • 使用

    USE 数据库名;

表操作

  • 查询当前数据库中所有表

    SHOW TABLES;

  • 查询表结构

    DESC 表名;

  • 查询指定表的建表语句

    SHOW CREATE TABLE 表名;

  • 创建表

    CREATE TABLE 表名（

    ​ 字段1 字段1类型 [COMMENT 字段1注释]，

    ​ 字段2 字段2类型 [COMMENT 字段2注释]，

    ​ ……

    ​ 字段n 字段n类型 [COMMENT 字段n注释]

    )[COMMENT 表注释];

  • 添加字段

    ALTER TABLE 表名

    ADD 字段名 类型(长度)

    [COMMENT 注释 ] [约束];

  • 修改数据类型

    ALTER TABLE 表名

    MODIFY 字段名 新 数据类型(长度);

  • 修改字段名和字段类型

    ALTER TABLE 表名

    CHANGE 旧字段名 新字段名 类型(长度)

    [COMMENT 注释 ] [约束];

  • 删除字段

    ALTER TABLE 表名

    DROP 字段名;

  • 修改表名

    ALTER TABLE 表名

    RENAME TO 新表名;

  • 删除表

    DROP TABLE [IF EXISTS] 表名;

  • 删除指定表，并重新创建该表–即清除表中数据（格式化）

    TRUNCATE TABLE 表名;

索引

  • 创建索引

    CREATE [UNIQUR] [CLUSTER] INDEX 索引名

    ON 表名(列名1 [次序],列名2 [次序])

    –次序：升序ASC，降序DESC

  • 删除索引

    DROP INDEX 索引名 ON 表名

1、原理

1.1什么是WEB

WEB是建立在Internet上的一种网络服务，为浏览者在Internet上查找和浏览信息提供了图形化的、易于访问的交互页面。

1.2 WEB技术分类

1.2.1前端

前端就是我们平常上网所看到的东西，包括了页面如何布局、如何上色、如何输入、如何点击等等。通俗的讲，前端就是在前面的，看得见的。

前端必学的三个技术：

1.HTML(超文本标记语言)，不是一门编程语言，而是用于定义内容结构的标记语言。

2.CSSCSS 也不是真正的编程语言，甚至不是标记语言。用于渲染web页面，

3.JavaScript是一种脚本，一门编程语言，它可以在网页上实现复杂的功能，网页展现给你的不是简单的静态信息，而是实时的内容更新，交互式的地图，2D/3D 动画，滚动播放的视频等等。

Web前端是指Web开发中负责实现Web界面和用户交互的技术领域，通常指网站或Web应用程序中运行在用户设备（如计算机、手机、平板电脑）上的前端部分，包括网页的设计、布局、样式、交互和效果等方面。

Web前端技术主要包括以下几个方面：

  1. HTML：负责Web页面的结构和内容的描述，是Web前端技术的基础。

  2. CSS：负责Web页面的样式和布局的描述，包括字体、颜色、边框、背景等方面。

  3. JavaScript：负责Web页面的交互和效果的实现，包括事件处理、动画效果、表单验证等方面。

  4. 前端框架和库：为Web前端开发提供了更高效、更简单、更规范的开发方式，如Vue.js、React、jQuery等。

  5. 前端工具：为Web前端开发提供了更好的开发体验和效率，如Webpack、Gulp、Grunt等。

1.2.2中端

Web中端通常是指Web应用程序中的业务逻辑处理部分，位于Web前端和Web后端之间，主要负责Web应用程序的业务逻辑处理和中间件的开发。

Web中端技术的发展，主要是为了解决Web应用程序中业务逻辑处理越来越复杂的问题。随着Web应用程序的快速发展和功能日益复杂，Web前端和Web后端已经无法满足业务逻辑处理的需求，因此Web中端技术应运而生。

Web中端技术主要包括以下几个方面：

  1. 业务逻辑处理：负责Web应用程序的业务逻辑处理，如数据处理、计算、验证、数据交互等。

  2. 中间件开发：负责处理Web应用程序中的中间件，如消息队列、缓存、搜索、分布式等。

  3. Web服务开发：负责Web应用程序的接口开发，如RESTful API、SOAP等。

  4. 分布式系统开发：负责Web应用程序中的分布式系统开发和管理。

Web中端技术的发展趋势是面向分布式和微服务架构，将Web应用程序中的业务逻辑处理和中间件拆分成独立的服务，实现高可用、高性能和高扩展性。同时，Web中端技术也需要具备一定的前端和后端技术，与Web前端和Web后端协同工作，共同构建出一个完整的Web应用程序。

1.2.3后端

后端是在后台工作的，控制着前端的内容，主要负责程序设计架构思想，管理数据库等。涉及动态语言如PHP、ASP、JSP等。

Web后端是指Web开发中负责处理服务器端逻辑和数据处理的技术领域，主要包括网站或Web应用程序中运行在服务器端的后端部分，用于处理用户请求、存储数据、提供API等功能。

Web后端技术主要包括以下几个方面：

  1. 服务器端语言：Web后端的核心语言，用于处理请求和响应、连接数据库、实现业务逻辑等，如PHP、Java、Python、Node.js等。

  2. 数据库：Web后端需要用到的数据存储和查询，常用的关系型数据库有MySQL、Oracle、SQL Server等，常用的非关系型数据库有MongoDB、Redis等。

  3. Web框架和库：为Web后端开发提供了更高效、更简单、更规范的开发方式，如Django、Flask、Spring、Express等。

  4. Web服务器：用于接收和处理来自客户端的HTTP请求，将请求分发给后端程序处理，如Apache、Nginx等。

Web后端开发的目标是实现服务器端逻辑和数据处理功能，为Web前端提供数据和服务支持。在Web应用程序中，Web后端和Web前端协同工作，共同构建出一个完整的Web应用程序。由于Web应用程序的复杂性和高并发性，Web后端开发也成为了Web开发中的重要领域。

3、注册表

3.1什么是注册表？

注册表是windows操作系统、硬件设备以及客户应用程序得以正常运行和保存设置的核心“数据库”，也可以说是一个非常巨大的树状分层结构的数据库系统。

注册表中记录了用户安装在计算机上的软件和每个程序的相关信息，通过它可以控制硬件、软件、用户环境和操作系统界面的数据信息文件。

注册表文件的数据信息保存在system.dat和user.dat中、利用regedit.exe程序能够存取注册表文件。

3.2注册表编辑器

3.2.1编辑器说明

win+r调出运行窗口，输入regedit进入注册表编辑器。

根键： 注册表计算机下的分支，可以理解为文件夹，以HKEY座位前缀
项： 可以理解为根键的子文件夹，分父项和子项，包含值。
值： 由名称、数据类型和数据三部分组成

3.2.2注册表五大根键

注：五大根键实际上平不是五个，HKEY_LOCAL_MACHINE、HKEY_USERS这两个才是真正的注册表键，其它都是从某个分支映射出来的，

HKEY_CLASSES_ROOT 该根键包括启动应用程序所需的全部信息，包括扩展名，应用程序与文档之间的关系，驱动程序名，DDE和OLE信息，类ID编号和应用程序与文档的图标等。
HKEY_CURRENT_USER 根键包括当前登录用户的配置信息，包括环境变量，个人程序以及桌面设置等
HKEY_LOCAL_MACHINE 根键包括本地计算机的系统信息，包括硬件和操作系统信息，安全数据和计算机专用的各类软件设置信息
HKEY_USERS 根键包括计算机的所有用户使用的配置数据，这些数据只有在用户登录系统时才能访问。这些信息告诉系统当前用户使用的图标，激活的程序组，开始菜单的内容以及颜色，字体
HKEY_CURRENT_CONFIG 根键包括当前硬件的配置信息，其中的信息是从HKEY_LOCAL_MACHINE中映射出来的。

4、防火墙

4.1定义和工作原理

定义：是一种计算机网络安全系统，可以限制进出网络的互联网流量。

工作原理：过滤掉异常或不受信任的流量，允许正常或受信任的流量通过。
